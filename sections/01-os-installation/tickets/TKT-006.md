# TKT-006: New starter requires a local user account created and verified before first day

**Status:** Resolved  
**Priority:** High  
**System:** Freshdesk

---

## Resolution Steps
1. Obtained the new starter's name and device details from the requester
2. Created a new local user account via Settings → Accounts → Other Users → Add account → Add a user without a Microsoft account
3. Set a temporary password and advised the requester to share it securely with the new starter
4. Signed out of the current session and logged into the new account to confirm it initialised correctly

> **Note:** Forcing a password change on next login is an Active Directory feature and is not available for local accounts. In a domain environment this would be configured via AD Users and Computers.
