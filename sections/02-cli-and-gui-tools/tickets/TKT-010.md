# TKT-010: User unable to reach internal resources — suspected DNS issue

**Status:** Resolved
**Priority:** High
**System:** Freshdesk

---

## Resolution Steps
1. Ran `ping google.com` — failed, then `ping 8.8.8.8` — succeeded, confirming DNS rather than connectivity was the issue
2. Ran `nslookup google.com` to confirm DNS resolution was failing
3. Ran `ipconfig /flushdns` to clear the local DNS cache
4. Retested with `ping google.com` — resolved successfully
5. Confirmed the user could access internal resources
